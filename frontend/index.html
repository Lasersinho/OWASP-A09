Log Forging Detected

OWASP A09:2021 — Security Logging and Monitoring Failures

Riesgo Alto (Critical)

Cuando una aplicación escribe en el sistema de logs información proporcionada por el usuario sin neutralizar los saltos de línea (\n, \r), un atacante puede fabricar registros falsos que corrompen la auditoría del sistema (SIEM).

{} Comparativa de Código

Vulnerable (app.py) 
def login():
    username = request.json.get('username')
    # ❌ El input fluye directo al archivo de logs
    logger.info(f"LOGIN_ATTEMPT user={username}")
    
    # ... chequeo fallido, pero el log de 
    # intento ya fue escrito y corrompido.
Seguro (Mitigado) 
def login():
    username = request.json.get('username')
    # ✅ Sanitizar saltos de línea primero
    sanitized = username.replace('\n', '\\n')\
                        .replace('\r', '')
                        
    logger.info(f"LOGIN_ATTEMPT user={sanitized}")
